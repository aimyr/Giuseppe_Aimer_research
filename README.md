# GIuseppe_Aimer_research
Large Language Models (LLMs) are increasingly deployed as evaluators in "LLM-as-a-judge" scenarios. While recent studies indicate that models tend to prefer their own output, it remains unclear whether this preference stems from objective quality improvements or an implicit "self-recognition" bias. This research investigates the causal link between an LLM's ability to detect AI-generated text and its evaluation of that text.
To decouple perceived quality from authorship, we utilize a dataset of human-written debate arguments. We generate parallel versions of these arguments using LLMs at varying levels of intervention: (1) light paraphrasing, (2) stylistic improvement, and (3) complete rewriting, while strictly preserving the original semantic meaning and stance. We then task the models with two objectives: identifying the authorship (AI vs. Human) and scoring the quality of the arguments.
By comparing scores assigned to human originals versus AI-edited variations, we aim to determine if LLMs exhibit an "in-group bias"—systematically favoring text they recognize as synthetic, even when semantic content remains constant. This study addresses a critical safety concern: if LLMs possess a "survival strategy" to blindly favor AI-generated patterns, their utility as neutral evaluators is fundamentally compromised.

# Dataset Structure and File Organization

This project contains two datasets, organized by task structure and evaluation type.

- The first dataset consists of **debate arguments** and is designed for argument comparison tasks.
- The second dataset consists of **paraphrased research abstracts** and is designed for abstract comparison tasks.

Both datasets are divided into two main categories:

- **Self Recognition**
- **Scoring**

---

## 1. Self Recognition

The **Self Recognition** category evaluates whether a model can identify text that it believes it generated itself.

This category contains two subtypes.

### 1.1 `self_recognition_with_one_abstract` / `self_recognition_with_one_argument`

In this setting, the model is given a single abstract or argument and asked whether it believes it wrote that text.

#### Example
`arguments_1_dataset/self_recognition_with_one_argument/self_rec_one_gpt5nano.csv`

This file contains the CSV output of GPT-5 Nano for the task where the model was shown one argument at a time and asked whether it believed it had written it.

### 1.2 `self_recognition_by_preference_from_pair`

In this setting, the model is given two candidate texts and asked which of the two it believes it wrote.

#### Example
`arguments_1_dataset/self_recognition_by_preference_from_pair/self_rec_preference_gemini.csv`

This file contains Gemini’s answers for the pairwise self recognition task.

---

## 2. Scoring

The **Scoring** category evaluates how models judge the quality of texts.

This category contains three subtypes.

### 2.1 `one_abstract_scoring` / `one_argument_scoring`

In this setting, the model evaluates one abstract or one argument at a time and assigns a score.

This can be done in two ways:

- the model scores the text based on its own judgment
- the model scores the text using instructions or a rubric written by the researcher

#### Example 1
`arguments_1_dataset/scoring/one_argument_scoring/scoring_llama_instruct.csv`

This file contains LLaMA’s scoring output when using the researcher’s instructions.

#### Example 2
`arguments_1_dataset/scoring/one_argument_scoring/scoring_llama.csv`

This file contains LLaMA’s scoring output when scoring on its own, without the researcher-provided rubric.

### 2.2 `preference_between_2_abstracts` / `preference_between_2_arguments`

In this setting, the model is given two texts and asked to decide which one is better.

#### Example
`arguments_1_dataset/scoring/preference_between_2_arguments/preference_gpt4omini.csv`

This file contains GPT-4o Mini’s pairwise preference judgments between two arguments.

### 2.3 `one_abstract_scoring_with_rubric_instructions` / `one_argument_scoring_with_rubric_instructions`

In this setting, evaluation instructions or rubrics are first generated by different models.

Then, each model evaluates individual abstracts or arguments using the rubrics created by other models.

#### Example
`arguments_1_dataset/scoring/one_argument_scoring_with_rubric_instructions/rubric_gemini/scores_gemini-2_5-flash_llama`

This folder contains CSV files where Gemini produced the answers while using rubric instructions associated with another model.

The LLM name in the folder or file name indicates which model generated the rubric or instructions.
